# VideoGet-v2 项目更新总结

## 📋 项目概述

**VideoGet-v2** 是一个视频下载工具，支持从多个平台下载视频，包括 YouTube、Bilibili、抖音等。

- **前端**：Vue 3 + Tailwind CSS + Vite
- **后端**：Python 3.11 + FastAPI + yt-dlp
- **支持的平台**：YouTube、Bilibili、抖音、快手、TikTok、Instagram、Twitter/X、Facebook、Vimeo、Twitch 等 1000+ 平台

## 🔄 项目更新历史

### 1. 项目初始化与环境配置

- **克隆项目**：从 GitHub 导入项目到本地
- **安装依赖**：
  - 前端：`npm install`
  - 后端：`pip install -r requirements.txt`
- **安装 ffmpeg**：使用 winget 安装 ffmpeg（Windows 系统）

### 2. 功能修复与改进

#### 2.1 依赖与环境问题修复

- **前端依赖问题**：解决 "vite' 不是内部或外部命令" 错误，通过运行 `npm install` 安装依赖
- **后端命令问题**：解决 "Python was not found" 错误，使用 `py` 命令代替 `python` 命令
- **Docker 配置问题**：由于 Windows 环境限制，采用本地开发方式

#### 2.2 核心功能修复

- **ffmpeg 集成**：
  - 初始配置：使用不需要 ffmpeg 的单格式策略
  - 最终方案：恢复使用 ffmpeg 合并最佳视频和音频的策略
  - 格式选择：`bestvideo[height<=xxx]+bestaudio/best[height<=xxx]`

- **画质选择功能**：
  - **前端显示**：修复了显示不标准画质选项（如 27p、45p、90p）的问题
  - **后端处理**：增强了对不同画质参数的处理能力
  - **格式映射**：建立了标准画质与分辨率范围的映射关系

- **下载逻辑优化**：
  - 增加了对 360p 画质的支持
  - 增加了对数字格式画质的处理
  - 增强了错误处理能力

#### 2.3 代码结构优化

- **移除调试信息**：清理了测试过程中添加的调试日志
- **测试文件清理**：删除了临时测试文件
- **代码规范**：保持了项目的代码风格和规范

#### 3.1 关键修改文件

1. **视频信息获取**：
   - 使用 yt-dlp 提取视频信息

   - 使用 ffmpeg 合并最佳视频和音频

   - 支持下载字幕

3. **任务管理**：
   - 异步任务处理
   - 下载进度跟踪
   - 任务状态管理

#### 3.3 技术创新

- **画质范围映射**：使用分辨率范围来归类不同质量的视频
- **智能格式选择**：优先选择包含音频的格式，确保下载的视频有声音
- **错误处理**：增强了对各种异常情况的处理能力

## 📝 使用说明

1. **启动前端**：
   ```bash
   cd frontend
   npm run dev
   ```

2. **启动后端**：
   ```bash
   cd backend
   py main.py
   ```

3. **访问应用**：
   - 前端：http://localhost:5173
   - 后端 API：http://localhost:8000/api

4. **使用方法**：
   - 在前端输入视频链接
   - 选择画质和其他选项
   - 点击"开始下载"按钮
   - 等待下载完成并自动打开下载链接
